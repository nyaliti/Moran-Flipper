
# Contributing to Moran Flipper v1.0

We welcome contributions to the Moran Flipper v1.0 project! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository on GitHub.
2. Clone your fork locally.
3. Create a new branch for your feature or bug fix.
4. Make your changes in your branch.
5. Test your changes thoroughly.
6. Commit your changes and push to your fork on GitHub.
7. Submit a pull request from your fork to the main repository.

## Coding Standards

- Follow the MQL5 coding style guide.
- Write clear, readable, and well-commented code.
- Ensure your code is properly indented and formatted.
- Use meaningful variable and function names.

## Testing

- Add appropriate unit tests for any new functionality.
- Ensure all existing tests pass before submitting a pull request.
- Include backtesting results for any changes to the trading strategy.

## Documentation

- Update the README.md file if your changes require it.
- Add inline comments to explain complex parts of your code.
- Update or add to the user documentation if necessary.

## Submitting Changes

1. Push your changes to your fork on GitHub.
2. Submit a pull request to the main repository.
3. Clearly describe your changes and their motivations in the pull request description.
4. Link any relevant issues in the pull request description.

## Code Review

- All submissions, including submissions by project members, require review.
- We use GitHub's pull request workflow to review and merge changes.

## Community

- Be respectful and considerate in all project interactions.
- Help others who have questions or issues.
- Share your knowledge and experiences with the community.

Thank you for contributing to Moran Flipper v1.0!
